# 視覺方向

這份草稿以目前「好理家在－財務健檢網」測驗頁為基準，延伸到獨立的情侶金錢默契測驗。它是本機預覽用的設計依據，Kevin 確認前不視為正式規範。

## 原則

- 先說明用途、流程與結果，再讓使用者進入測驗。
- 整體感受溫暖、安心、不評分、不像投資商品或心理診斷。
- 使用白色頁面、淡粉與淡橘區塊、深棕文字、粉橘主按鈕。
- 題目卡維持寬鬆留白，答案使用白色圓角框，不以強烈顏色造成答題暗示。
- 觸控裝置只有在使用者真的點選答案後才顯示粉紅選取色；hover 回饋只提供給有滑鼠的裝置，避免 iPhone 黏住底色造成預選錯覺。
- 保留原本六個評估面向與題意；夥伴提供的生活例子只用來校準口語與具體程度，不能直接取代旅行、消費、投資風險或共同決策等原始構念。
- 題幹可以用鞋子、3C、旅行房型等熟悉事物幫助想像，但不把一種物品、品牌或特定伴侶的做法當成所有人的標準情境。
- 每個選項先放可掃讀的短標籤，再補一句差異；若都不符合，直接提供「都不太像，先略過」。
- 手機優先，中文自然換行，主要按鈕在 360～430 px 寬度可直接點擊。

## 色彩

- 題目文字：`#5F4B66`
- 主按鈕：`#F4A46F`
- 主按鈕 hover：`#EB925B`
- 品牌粉：`#FF9A9E`
- 淡粉背景：`#FFF1F2`
- 淡橘背景：`#FFF4EC`
- 淡綠輔助：`#EAF8F5`
- 卡片／頁面：`#FFFFFF`
- 邊線：`rgba(155, 142, 163, 0.18)`

## 字體與層級

- 字體：優先使用台灣系統中文字體與 `Noto Sans TC`，不新增外部字型依賴。
- 首頁標題：桌機約 40～48 px；手機自然縮到 34～38 px。
- 區塊標題：28～36 px。
- 題目：20～24 px，粗體。
- 內文：16～18 px，行高 1.7 左右。

## 元件

- 導覽列：白底、淡粉陰影、左側品牌標誌、右側「測驗說明」。
- 主按鈕：粉橘底、白字、12 px 圓角、柔和橘色陰影。
- 次按鈕：白底、粉橘邊線與文字。
- 資訊標籤：白色膠囊、淡灰粉邊線、可搭配簡單 emoji。
- 卡片：16～24 px 圓角，淡粉邊線與低彩度陰影。
- 進度條：淡粉軌道、品牌粉進度。
- 結果頁：只呈現本次分數與當前類型；完整級別與類型規則保留在內部計分與產品檢核，不放在參與者報告。
- 成長追蹤：搬進好理家在平台後，由登入會員主動將本次日期、總分、類型、六軸分數與選填的三個月成長目標保存到個人中心；不傳逐題答案、配對碼或私密提醒。3 個月後重測時，由會員 API 讀回歷次紀錄並顯示變化。
- 服務推薦：七項正式入口全部保留，但依作答完整度、風險提醒、關係類型與最低面向，只突出一個主要下一步與兩個次要選項；其餘四項預設收合，外連另開分頁。

## 流程

1. 測驗說明頁：用途、所需時間、怎麼進行、會看到什麼、隱私提醒。
2. 模式選擇頁：兩支手機各自測／同一台手機輪流測。
3. 12 題核心測驗，完成後直接進入配對或同機交接。
4. 配對或同機交接。
5. 共同報告、可以參考的對話主題、關係成長追蹤與「好理家在還可以」服務推薦。

## 驗證

- 桌機：說明頁、模式頁、題目頁、配對碼頁、報告頁。
- 手機：360、393、430 px 無水平溢位、無截斷按鈕。
- 真人流程：說明頁 → 模式頁 → 作答 → 配對 → 報告。
